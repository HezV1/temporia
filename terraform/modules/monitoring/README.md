# Monitoring Module — `temporia-archive`

Provisions the Log Analytics workspace that backs the purple-team detection loop.

## What it creates

| Resource | Name | Notes |
|----------|------|-------|
| Log Analytics workspace | `temporia-archive` | SKU `PerGB2018`, 30-day retention |

## Why it exists

The Lexmechanic archives everything. This workspace is where logs and telemetry land so the **skitarii** (blue) workloads can write detections against real attack data generated by the **myrmidon** (red) workloads. 30-day retention bounds cost while keeping a useful history window.

In later sessions this workspace becomes the sink for AKS diagnostic settings and the `lexmechanic` namespace's log pipeline.

| `resource_group_name` | string | — | Required |
| `location` | string | — | Required |

## Outputs

`workspace_id`, `workspace_name`, `workspace_customer_id`.
